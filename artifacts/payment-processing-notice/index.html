Pago en procesamiento como estado seguro del checkout.

Usar los prints como referencia visual y mantener el documento simple: el cambio propuesto es pasar de un notice local a un estado global de checkout, con salida obligatoria ante error o timeout.

Failsafe obligatorio

Si el user vuelve al checkout por error, el loading tiene que desaparecer.

En ningún caso esta mejora puede dejar al user preso en loading infinito. Si la order falla, si el checkout re-renderiza con error, si hay back navigation, reload o timeout, el estado global de processing debe ser removido.

  • Ante error de order/payment: remover loading y mostrar el error recuperable.
  • Ante timeout sin resultado: salir del loading y devolver control. Valor inicial para revisar con Engineering: 10 segundos.
  • El estado de processing no debe persistir por cache, reload o retorno al checkout.
